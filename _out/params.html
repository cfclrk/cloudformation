DeploymentName

A deployment is a complete deployed application, potentially comprised of many CloudFormation stacks. This is sometimes called an "environment", but that is an overloaded term. Use the DeploymentName to indicate which deployment a Stack is part of. 

  DeploymentName:
    Type: String
    Description: A name for this deployment

KeyName

AWS does not provide a built-in CloudFormation type for EC2 KeyPairs. This was a deliberate choice: a KeyPair consists of a public and private key, and AWS does not want to expose the private key in CloudFormation outputs.

This is one example where Terraform has an advantage: since Terraform runs locally, it can create a KeyPair and save the private key locally.

So, in CloudFormation, you have two options:

  1. Create the EC2 KeyPair beforehand. Then, provide the existing KeyName to your CloudFormation template as a parameter.
  2. Define a CloudFormation Custom Resource (i.e. a Lambda Function) in your CloudFormation template. The Custom Resource could create the KeyPair and save the private key somewhere (like SecretsManager or SSM Parameter Store).
  KeyPair:
    Type: AWS::EC2::KeyPair::KeyName
    Description: The name of an EC2 KeyPair
